II. The optimization problem

Logistic regression consists in finding the parameter $x \in \mathbb{R}^d$ that best fits: $$ b_i = \begin{cases} \phantom{-}1 \ \textrm{ if }\ \langle a_{i}, x\rangle \ge 0 \\ -1 \ \textrm{ if }\ \langle a_{i}, x\rangle < 0 \end{cases} $$ for all the samples $i$ of the training set (where $a_i$ is the $i^{th}$ line of $A$ and $b_i$ the $i^{th}$ element of $b$). To measure this fit, logistic regression chooses the following loss:

$$ \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) = \log \left(1+\exp \left(-b_{i}\left\langle a_{i}, x\right\rangle\right)\right) $$

The corresponding optimization problem we will consider is the following:

Minimization of the empirical risk $f$ of the $L_2$-regularized logistic regression model, on the dataset $(A_{\textrm{train}},b_{\textrm{train}})$: $$ \begin{aligned} \textrm{min}_{x\in \mathbb{R}^d} \ f(x) \ &= \ \textrm{min}_{x\in \mathbb{R}^d} \ \frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} f_i(x) \\ &= \ \textrm{min}_{x\in \mathbb{R}^d}\frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) + \frac{l_2}{2}\|x\|^{2}_2, \end{aligned} $$ where $l_2 \ge 0$ is the $L_2$ regularization coefficient.

To simplify the notation, we'll stop indexing $A$, $b$ and $n$ with "$\textrm{train}$" in the following.

Question: For any square matrix $M$, denote $\lambda_{\max}(M)$ the largest eighenvalue of $M$.
  • Prove that $\lambda_{\max}(\nabla^2 f_i) \le \lambda_{\max}(\frac{a_i a_i^T}{4} + l_2 I_d)$
  • Prove that $\lambda_{\max}(a_i a_i^T) \le \|a_i\|^2_2$

Answer:

We have $a_ia_i^T = \|a_i\|_2^2$. Let's take $x$ such that $a_ia_i^Tx = \lambda x$, $\lambda$ a real. Then :

\begin{aligned}a_ia_i^Tx = \lambda x &\implies \|a_i\|_2^2x = \lambda x \\ &\implies (\|a_i\|_2^2 - \lambda)x = 0 \\ &\implies \begin{cases} \lambda = \|a_i\|_2^2 \ \textrm { if x } \neq 0 \\ \lambda = 0 \ \textrm{if x = 0} \end{cases} \\ &\implies \lambda_{max}(a_ia_i^T) \leq \|a_i\|_2^2 \end{aligned}

A consequence of these results is that the $f_i$s and $f$ are all $L$-smooth with $L \le \frac{\max_i\|a_i\|^2_2}{4} + l_2$ and we can therefore use the following value in our implementation:

L = 0.25 * max(np.linalg.norm(A,2,axis=1))**2 + self.l2

def grad(x, i=None):
    '''
    Gradient of the objective function at x.
    If no i is given, grad should return the batch gradient.
    Otherwise, it should return the partial gradient associated the the datapoint (A[i], b[i]).
    '''

    if i is None: # return batch gradient
        output = -b/(1 + np.exp(b*np.dot(A, x)))
        output = output/n @ A
        output += self.l2*x
    else: # return partial gradient associated the datapoint (A[i], b[i])
        Ai, bi = A[i].reshape(1, -1), b[i]
        output = -bi/(1 + np.exp(bi*np.dot(Ai.reshape(1,-1), x)))
        output = output @ Ai
        output += self.l2*x
    return output

GD (Gradient descent)¶

At iteration k of the GD algorithm, the following update is executed:

$$ x_{k+1} \leftarrow x_{k} - \gamma\nabla f(x_k) $$
Question:
  • What stepsize $\gamma$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (3) and TODO (4)
  • Add your code to \lib\solvers.py.

Answer:

Assuming that the function is convex, differentiable (which is the case here) and L-smooth, the stepsize \begin{align*} \gamma = \frac{1}{L} \end{align*}

ensures the convergence of the Gradient Descent algorithm.

def GD(x0, grad, max_iter, n, L, mu):
    '''
    Parameters
    ----------
    x0: array, shape (nb_features,)
        Initialisation of the solver
    grad: function
        Gradient of the objective function
    max_iter: int
        Number of iterations (i.e. number of descent steps). Note that for GD or SVRG, 
        one iteration is one epoch i.e, one pass through the data, while for SGD, SAG and SAGA, 
        one iteration uses only one data point.
    n: int
        Dataset size
    L: float
        Smoothness constant of the objective function
    mu: float
        Strong convexity constant of the objective function

    Returns
    -------
    x: array, shape (nb_features,)
        final iterate of the solver
    x_tab: array, shape (nb_features, max_iter)
        table of all the iterates
    '''
    stepsize  = 1/L
    x         = x0
    x_tab     = np.copy(x)

    for k in range(max_iter):
        x     = x - stepsize*grad(x)
        x_tab = np.vstack((x_tab, x))

    return x, x_tab

SGD (Stochastic Gradient Descent)¶

At iteration k of the SGD algorithm, the following update is executed:

$$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow x_{k} - \gamma_k \nabla f_i(x_k) \\ \end{cases} $$
Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (5)
  • Add your code to \lib\solvers.py.

Answer:

A standard using the SGD algorithm is to take diminishing step-size to ensure convergence. So for this algorithm, we took \begin{align*} \gamma_k = \frac{1}{k+1} \end{align*} under the assumption that f is convex.

import random as rd

def SGD(x0, grad, prox, max_iter, n, L, mu):

    x         = x0
    x_tab     = np.copy(x)

    for k in range(max_iter):

        stepsizek = 1/(k+1)
        i = rd.randint(0, n-1)
        x = x - stepsizek*grad(x, i)
        if k%n == 0: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SAGA (Stochastic Average Gradient "Amélioré")¶

SAGA starts by computing a single batch gradient, then at iteration k, the following update is executed:

$$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow x_{k} - \gamma_k\big(\nabla f_i(x_k) - \alpha_i + \bar{\alpha}\big) \\ \alpha_i \leftarrow \nabla f_i(x) \end{cases} $$

where $(\alpha_j)_{j=1}^n$ are gradients evaluated at previous iterates and $\bar{\alpha}=\frac{1}{n}\sum^n_{j=1}\alpha_j$.

Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (6)
  • Add your code to \lib\solvers.py.

Answer:

Assuming that the $f_i$ are L-smooth and that f is stronglyx convex, a constant step-size of value \begin{align*} \gamma_k = \gamma = \frac{1}{2(\mu n + L)} \end{align*}

ensures convergence for the SAGA algorith.

def SAGA(x0, grad, max_iter, n, L, mu):

    x         = x0
    x_tab     = np.copy(x)
    stepsize = 1/(2*(mu*n + L))
    ai = 0
    asum = 0
    ng = 0
    for k in range(max_iter):

        i = rd.randint(0, n-1)
        g = grad(x, i)
        ab = asum / ng if ng != 0 else 0
        x = x - stepsize*(g - ai + ab)
        ai = g
        asum += g
        ng += 1
        if k%n == 0: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SVRG (Stochastic Variance Reduced Gradient)¶

At iteration k of the SVRG algorithm, we perform the following:

$$ \begin{cases} \textrm{Compute the batch gradient } \nabla f(x_k)\\ \textrm{Initialise }y_0 = x_k\\ \textrm{for m in 0...M-1: } \begin{cases} i \textrm{ chosen randomly in } \{1...n\} \\ y_{m+1} \leftarrow y_{m} - \gamma_k\big(\nabla f_i(y_m) - \nabla f_i(x_k) + \nabla f(x_k)) \end{cases}\\ x_{k+1} \leftarrow y_{M} \end{cases} $$
Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (7)
  • Add your code to \lib\solvers.py.

Answer:

A result from the course states that, with f convex, any sequence converging toward > 0 whose serie does not converge is a correct value for $\gamma_k$. Experiments showed that the following value is a good one : \begin{aligned} \gamma_k = \frac{1}{(k+1)^{3/4}} \end{aligned}

def SVRG(x0, grad, max_iter, n, L, mu, M=10):

    x         = x0
    x_tab     = np.copy(x)

    for k in range(max_iter):

        gf = grad(x)
        ym = x
        stepsize = 1/(k+1)**(3/4)
        for m in range(M):
            i = rd.randint(0,n-1)
            gi = grad(x, i)
            ym = ym - stepsize*(grad(ym, i) - gi + gf)
        x = ym
        x_tab = np.vstack((x_tab, x))

    return x, x_tab

Question:
  • Rewrite the empirical risk minimization objective with both $L_1$ and $L_2$ regularization (with coefficients $l_1$ and $l_2$).
  • Is this objective $\mu$-strongly convex, $L$-smooth, differentiable?
  • Complete the TODO (8) to account for the new term in the objective.
  • Add your code to lib/linear_model.py.

Answer: The empirical risk minimization objective with both $L_1$ and $L_2$ regularization is : \begin{aligned} \textrm{min}_{x\in \mathbb{R}^d} \ f(x) &= \ \textrm{min}_{x\in \mathbb{R}^d}\frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) + \frac{l_2}{2}\|x\|^{2}_2 + l_1|x| \end{aligned}

This objective is not differentiable due to the presence of the $L_1$ norm. It is only stricly convex since none of its composite is strongly convex.

self._empirical_risk = lambda x: sum(self._logistic_loss(A, b, x))/n \
                                 + self.l2/2.0 * np.linalg.norm(x,2) \
                                 + self.l1 * np.linalg.norm(x,1)

The minimization algorithms we've implemented have a slow rate of convergence for this new objective function. We will therefore use proximal variations of these algorithms.

Question: Using the $\textrm{prox}_{l_1 \|\cdot\|_1}$ operator, write down the $k^{th}$ step of proximal GD, proximal SGD, proximal SAGA, and proximal SVRG.

Answer:

The $k^{th}$ step of each algorithm using the proximal operator does > not change by much:

For GD : $$ x_{k+1} \leftarrow \textrm{prox}_{\gamma l_1 \|\cdot\|_1}(x_{k} - \gamma\nabla f(x_k)) $$ For SGD : $$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow \textrm{prox}_{ \gamma_k l_1 \|\cdot\|_1}(x_{k} - \gamma_k \nabla f_i(x_k)) \\ \end{cases} $$ For SAGA : $$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow \textrm{prox}_{\gamma_k l_1 \|\cdot\|_1}(x_{k} - \gamma_k\big(\nabla f_i(x_k) - \alpha_i + \bar{\alpha}\big)) \\ \alpha_i \leftarrow \nabla f_i(x) \end{cases} $$

For SVRG : $$ \begin{cases} \textrm{Compute the batch gradient } \nabla f(x_k)\\ \textrm{Initialise }y_0 = x_k\\ \textrm{for m in 0...M-1: } \begin{cases} i \textrm{ chosen randomly in } \{1...n\} \\ y_{m+1} \leftarrow \textrm{prox}_{\gamma_k l_1 \|\cdot\|_1}(y_{m} - \gamma_k\big(\nabla f_i(y_m) - \nabla f_i(x_k) + \nabla f(x_k))) \end{cases}\\ x_{k+1} \leftarrow y_{M} \end{cases} $$

Question:
  • Give a closed-form expression of $\textrm{prox}_{l_1 \|\cdot\|_1}$
  • Complete TODO (9) (note that you can access the $l_1$ coefficient using self.l1).
  • Add your code to \lib\linear_model.py.

Answer:

By definition, we have : \begin{aligned} prox_{l_1 \|\cdot\|_1}(x) &= argmin_y (l_1 \|y\|_1 + \frac{1}{2}\|y - x\|_2^2) \\ &= argmin_y ( l_1 \sum_i |y_i| + \frac{1}{2} \sum_i (y_i - x_i)^2 )\\ &= argmin_y \sum_i(l_1 |y_i| + \frac{1}{2} (y_i - x_i)^2)\\ &= \{prox_{l_1 \|\cdot\|_1}(x)_i\}_i \end{aligned}

with $prox_{l_1 \|\cdot\|_1}(x)_i$ = $ argmin_{y_i} (l_1 |y_i| +\frac{1}{2} (y_i - x_i)^2)$

Then, if $y_i > 0$, the derivative is ($y_i - x_i + l_1)$ so $y_i = x_i - l_1$ if $x_i > l_1$. Similarly, if $y_i < 0$, $y_i = l_1 + x_i$ if $x_i < -l_1$. So :

\begin{aligned} prox_{l_1 \|\cdot\|_1}(x) &= \{sgn(x_i)max(|x_i| - l_1, 0)\}_i \end{aligned} 
def prox(x, stepsize):
    return np.sign(x)*np.maximum(np.abs(x) - stepsize*self.l1, 0)

from torch.optim.optimizer import Optimizer, required
from torch import zeros
import copy

class SAGA(Optimizer):
    """Implementation of SAGA optimizer
    """

    def __init__(self, params, lr=required):
        defaults = dict(lr=lr)
        super(SAGA, self).__init__(params, defaults)
        self.state["iteration"] = 0

    def __setstate__(self, state):
        super(SAGA, self).__setstate__(state)

    def step(self, closure=None):
        """ Performs a single optimization step.
        Arguments:
            closure (callable, optional): A closure that reevaluates the model
                and returns the loss.
        """
        loss = None
        if closure is not None:
            loss = closure()
            

        for group in self.param_groups:
            for p in group['params']:
                if p.grad is None:
                    continue
                d_p = p.grad.data
                param_state = self.state[p]
                if 'gradient_sum' not in param_state:
                    param_state['gradient_sum'] = d_p.data.clone()
                    gs = torch.zeros_like(d_p)
                else:
                    ngs = param_state['gradient_sum']
                    gs =  ngs.data.clone()
                    ngs.add_(d_p.data)
                    
                if 'last_gradient' not in param_state:
                    lg = torch.zeros_like(d_p)
                else:
                    lg = param_state['last_gradient']
                param_state['last_gradient'] = d_p.data.clone()
                
                p.data.add_(-group['lr']*d_p)
                p.data.add_(-group['lr']/(self.state["iteration"]+1)*gs)
                p.data.add_(+group['lr']*lg)

        self.state["iteration"] += 1        
        
        return loss

        self.dropout2 = nn.Dropout2d(0.5)
        self.fc1 = nn.Linear(9216, 128)
        self.fc2 = nn.Linear(128, 10)

    def forward(self, x):
        x = self.conv1(x)
        x = F.relu(x)
        x = self.conv2(x)
        x = F.max_pool2d(x, 2)
        x = self.dropout1(x)
        x = torch.flatten(x, 1)
        x = self.fc1(x)
        x = F.relu(x)
        x = self.dropout2(x)
        x = self.fc2(x)
        output = F.log_softmax(x, dim=1)
        return output

# The following code is for training neural networks

model = Neural_Network().to(device)
model.train()

# we are using stochastic gradient descent optimizer
optimizer = SAGA(model.parameters(), lr=0.01)

# we train the neural networks for 1000 epochs.
# epochs basically means the number of time the whole set of data will pass through the neural network.
def train(model, device, train_loader, optimizer, epoch):
    for batch_idx, (data, target) in enumerate(train_loader):
        data, target = data.to(device), target.to(device)
        optimizer.zero_grad()
        output = model(data)
        loss = F.nll_loss(output, target)
        loss.backward()
        optimizer.step()
        if batch_idx % 100 == 0:
            print('Train Epoch: {} [{}/{} ({:.0f}%)]\tLoss: {:.6f}'.format(
                epoch, batch_idx * len(data), len(train_loader.dataset),
                100. * batch_idx / len(train_loader), loss.item()))
        
def test(model, device, test_loader):
    model.eval()
    test_loss = 0
    correct = 0
    with torch.no_grad():
        for data, target in test_loader:
            data, target = data.to(device), target.to(device)
            output = model(data)
            test_loss += F.nll_loss(output, target, reduction='sum').item()  # sum up batch loss
            pred = output.argmax(dim=1, keepdim=True)  # get the index of the max log-probability
            correct += pred.eq(target.view_as(pred)).sum().item()

    test_loss /= len(test_loader.dataset)

    print('\nTest set: Average loss: {:.4f}, Accuracy: {}/{} ({:.0f}%)\n'.format(
        test_loss, correct, len(test_loader.dataset),
        100. * correct / len(test_loader.dataset)))
    
NUM_EPOCHS = 15
for epoch in range(NUM_EPOCHS):
    print(f"#--- EPOCH {epoch} - TRAIN")
    train(model, device, train_loader, optimizer, epoch)
    print(f"#--- EPOCH {epoch} - TEST")
    test(model, device, test_loader)
