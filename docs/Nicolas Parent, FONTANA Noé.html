II. The optimization problem

Logistic regression consists in finding the parameter $x \in \mathbb{R}^d$ that best fits: $$ b_i = \begin{cases} \phantom{-}1 \ \textrm{ if }\ \langle a_{i}, x\rangle \ge 0 \\ -1 \ \textrm{ if }\ \langle a_{i}, x\rangle < 0 \end{cases} $$ for all the samples $i$ of the training set (where $a_i$ is the $i^{th}$ line of $A$ and $b_i$ the $i^{th}$ element of $b$). To measure this fit, logistic regression chooses the following loss:

$$ \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) = \log \left(1+\exp \left(-b_{i}\left\langle a_{i}, x\right\rangle\right)\right) $$

The corresponding optimization problem we will consider is the following:

Minimization of the empirical risk $f$ of the $L_2$-regularized logistic regression model, on the dataset $(A_{\textrm{train}},b_{\textrm{train}})$: $$ \begin{aligned} \textrm{min}_{x\in \mathbb{R}^d} \ f(x) \ &= \ \textrm{min}_{x\in \mathbb{R}^d} \ \frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} f_i(x) \\ &= \ \textrm{min}_{x\in \mathbb{R}^d}\frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) + \frac{l_2}{2}\|x\|^{2}_2, \end{aligned} $$ where $l_2 \ge 0$ is the $L_2$ regularization coefficient.

To simplify the notation, we'll stop indexing $A$, $b$ and $n$ with "$\textrm{train}$" in the following.

Question: $f$ is $\mu$-strongly convex.
  • Give a lower bound on $\mu$.
  • Complete TODO (1) in \lib\linear_model.py.

Answer: Clearly, a lower bound is $\mu \geq l_2$.

We denote $\sigma\left(z_{i}\right)=\frac{1}{1+e^{-z_{i}}}$ and remark that its image is in $[0, 1]$. The Hessian of the logistic loss is $\vec{\nabla}^{2} \textrm{LogisticLoss}(x)=\sum_{i=1}^{m} x_{i} x_{i}^{T} \sigma(b_i, \left\langle a_{i}, x\right\rangle )\left(1-\sigma(b_i, \left\langle a_{i}, x\right\rangle )\right)$. Since it consists in the sum of positive semidefinite matrices multiplied by positive constants, it is positive semidefinite. Thus the logistic loss is convex which is equivalent to the $l_2$ strong convexity of $f$.

Question: Denote $\lambda_{\max}$ the largest eigenvalue of the real symmetric matrix $AA^T$.
  • Prove that $\lambda_{\max}(\nabla^2 f_i) \le \lambda_{\max}(\frac{a_i a_i^T}{4} + l_2 I_d)$
  • Prove that $\lambda_{\max}(a_i a_i^T) \le \|a_i\|^2_2$

Answer:

The expression of the second order derivative is $$ D^2f_i(x)_{k,l}= \frac{(-b_i a_i^k)(-b_i a_i^l)\exp\left(-b_{i}\left\langle a_{i}, x\right\rangle\right))}{\left(1+\exp\left(-b_{i}\left\langle a_{i}, x\right\rangle\right) \right)^2} + l_2 \delta_k^l $$ which reads $$ D^2f_i(x)_{k,l} = \left( a_i a_i^T \right)_{k,l} \frac{b_i^2 \exp\left(-b_{i}\left\langle a_{i}, x\right\rangle\right)}{\left(1+\exp\left(-b_{i}\left\langle a_{i}, x\right\rangle\right) \right)^2} + l_2 \delta_k^l $$ taking into account that $b_i= \pm 1$ and knowing that the function $x\mapsto \frac{x}{\left(1+x\right)^2}$ has maximum $\frac{1}{4}$ on $\left[0,+\infty\right[$ we find $$ \lambda_{\max}(\nabla^2 f_i) \le \lambda_{\max}(\frac{a_i a_i^T}{4} + l_2 I_d) $$ The matrix $a_i a_i^T$ is a rank $1$ matrix, its eigenvalue is thus given by its trace $\|a_i\|^2$. We can conclude $$ \lambda_{\max}(a_i a_i^T) \le \|a_i\|^2_2 $$

A consequence of these results is that the $f_i\,'s$ and $f$ are all $L$-smooth with $L \le \frac{\max_i\|a_i\|^2_2}{4} + l_2$ and we can therefore use the following value in our implementation:

L = 0.25 * max(np.linalg.norm(A,2,axis=1))**2 + self.l2

def grad(x, i=None):
    '''
    Gradient of the objective function at x.
    If no i is given, grad should return the batch gradient.
    Otherwise, it should return the partial gradient associated the the datapoint (A[i], b[i]).
    '''

    if i is None: # return batch gradient
        output = -b/(1 + np.exp(b*np.dot(A, x)))
        output = sum(np.diag(output))/n @ A
        output += self.l2*x
    else: # return partial gradient associated the datapoint (A[i], b[i])
        output = -b[i]/(1 + np.exp(b[i]*np.dot(A[i], x)))
        output = output * A[i]
        output += self.l2 * x
    return output

GD (Gradient descent)¶

At iteration k of the GD algorithm, the following update is executed:

$$ x_{k+1} \leftarrow x_{k} - \gamma\nabla f(x_k) $$
Question:
  • What stepsize $\gamma$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (3) and TODO (4)
  • Add your code to \lib\solvers.py.

Answer:

The following holds :

  • The gradient of the objective is Lipschitz-continuous with constant $L$
  • The objective is strongly convex with constant $\mu$
  • F is bounded below on an open set (it is positive)

It is then proven that

$ \gamma \leq \frac{2}{\mu + L}$ ensures the convergence.

def GD(x0, grad, prox, max_iter, n, L, mu, l1):
    '''
    Parameters
    ----------
    x0: array, shape (nb_features,)
        Initialisation of the solver
    grad: function
        Gradient of the objective function
    max_iter: int
        Number of iterations (i.e. number of descent steps). Note that for GD or SVRG, 
        one iteration is one epoch i.e, one pass through the data, while for SGD, SAG and SAGA, 
        one iteration uses only one data point.
    n: int
        Dataset size
    L: float
        Smoothness constant of the objective function
    mu: float
        Strong convexity constant of the objective function
        
    Returns
    -------
    x: array, shape (nb_features,)
        final iterate of the solver
    x_tab: array, shape (nb_features, max_iter)
        table of all the iterates
    '''
    stepsize  = 2/(mu+L)
    x         = np.copy(x0)
    x_tab     = np.copy(x)

    for k in range(max_iter):
        x     = x - stepsize * grad(x)
        if l1:
            x = prox(x, stepsize * l1)
        x_tab = np.vstack((x_tab, x))

    return x, x_tab

SGD (Stochastic Gradient Descent)¶

At iteration k of the SGD algorithm, the following update is executed:

$$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow x_{k} - \gamma_k \nabla f_i(x_k) \\ \end{cases} $$
Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (5)
  • Add your code to \lib\solvers.py.

Answer:

We have checked that :

  • The gradient of the objective is Lipschitz-continuous with constant $L$
  • The objective is strongly convex with constant $\mu$
  • F is bounded below on an open set

then Stochastic Gradient Descent with diminishing step sizes converge if the stepsizes verify $\sum_{k=1}^{\infty} \alpha_{k}=\infty$ and $\sum_{k=1}^{\infty} \alpha_{k}^{2}<\infty$.

For instance, one can take $$\alpha_{k}=\frac{\beta}{\gamma+k} \text { for some } \beta>\frac{1}{c \mu} \quad \text { and } \quad \gamma>0 \text { such that } \alpha_{1} \leq \frac{\mu}{L}$$ with $L \geq c \geq 0$

Cf. Optimization Methods for Large-Scale Learning, 2018

def SGD(x0, grad, prox, max_iter, n, L, mu, l1):
    
    x = np.copy(x0)
    x_tab = np.copy(x)

    beta = 1.5 / (mu*L)
    gamma = 1/(mu**2)

    for k in range(1, max_iter):
        #step_size = 0.1/(k+1)**0.6
        step_size = beta / (gamma + k)
        i = np.random.randint(0,n)
        x = x - step_size * grad(x,i)
        if l1:
            x = prox(x, step_size * l1)
        if k%n == 0: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SAGA (Stochastic Average Gradient "Amélioré")¶

SAGA starts by computing a single batch gradient, then at iteration k, the following update is executed:

$$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow x_{k} - \gamma_k\big(\nabla f_i(x_k) - \alpha_i + \bar{\alpha}\big) \\ \alpha_i \leftarrow \nabla f_i(x) \end{cases} $$

where $(\alpha_j)_{j=1}^n$ are gradients evaluated at previous iterates and $\bar{\alpha}=\frac{1}{n}\sum^n_{j=1}\alpha_j$.

Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (6)
  • Add your code to \lib\solvers.py.

Answer:

The following assumptions are satisfied :

  • The gradient of the objective is Lipschitz-continuous with constant $L$
  • The objective is strongly convex with constant $\mu$
  • F is bounded below on an open set which contains the iterates $w_k$

Then a stepsize $\gamma_k = \frac{1}{2(\mu n + L)}$ ensures convergence.

Possible trick to use less memory

In SAGA, Defazio 2014, the authors highlight a possible implementation to reduce the memory overhead caused by the stored gradients. In the case logistic of regression, the derivative $f_i$ can be decomposed as a weighting coefficients and the problem data. We then only have to store the weighting coefficients. Indeed for $i \in [\![1;n]\!] $, we have :

$$f_i(x) = \log(1 + \exp(-b_i \langle a_i, x \rangle) \text{ and }f_i'(x) = -b_i a_i \frac{1}{1 + \exp(b_i \langle a_i, x \rangle)}$$

Thus we only need to store the value of $c_i = \frac{1}{1 + \exp(b_i \langle a_i, x \rangle)}$ for each $f_i'$.

def SAGA(x0, grad, prox, max_iter, n, L, mu, l1):

    d = x0.size
    x = np.copy(x0)
    x_tab = np.copy(x0)
    alpha = np.zeros((n,d))
    step = 0.5/(mu*n+L) 
    for k in range(n):
        alpha[k] = grad(x0,k)
    mean_alpha = np.mean(alpha,axis=0) 
    for k in range(n,max_iter):
        i = np.random.randint(0,n)
        new_grad = grad(x,i)
        g = new_grad - alpha[i] + mean_alpha
        mean_alpha = mean_alpha - 1/n * (alpha[i] - new_grad)
        alpha[i] = new_grad
        x = x - step * g
        if l1:
            x = prox(x, step * l1)
        if k%n == 0 and k!=n: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SVRG (Stochastic Variance Reduced Gradient)¶

At iteration k of the SVRG algorithm, we perform the following:

$$ \begin{cases} \textrm{Compute the batch gradient } \nabla f(x_k)\\ \textrm{Initialise }y_0 = x_k\\ \textrm{for m in 0...M-1: } \begin{cases} i \textrm{ chosen randomly in } \{1...n\} \\ y_{m+1} \leftarrow y_{m} - \gamma_k\big(\nabla f_i(y_m) - \nabla f_i(x_k) + \nabla f(x_k)) \end{cases}\\ x_{k+1} \leftarrow y_{M} \end{cases} $$
Question:
  • What stepsize $\gamma_k$ ensures the convergence of this algorithm and under what assumptions?
  • Complete TODO (7)
  • Add your code to \lib\solvers.py.

Answer:

Those conditions hold :

  • The gradient of the objective is Lipschitz-continuous with constant $L$
  • The objective is strongly convex with constant $\mu$
  • F is bounded below on an open set

We consider cycle of $m$ inner updates of step size $\alpha$.

The conditions of SVRG to converge is that $$\rho:=\frac{1}{1-2 \alpha L}\left(\frac{1}{m c \alpha}+2 L \alpha\right)<1$$ with $L \geq c \geq 0$

Cf. Optimization Methods for Large-Scale Learning, 2018

Note on convergence

We decide to update $x_{k+1}$ with the value $y_m$ even though $x_{k+1}$ is chosen randomly in the original convergence proof.

def SVRG(x0, grad, prox, max_iter, n, L, mu, l1):
    d = x0.size
    x = np.copy(x0)
    x_tab = np.copy(x0)
    alpha = np.zeros((n,d))
    
    mean_alpha = np.sum(alpha,axis=0)/n
    
    c = L
    step = 0.005
    m = 100
    print((1/(m*c*step) + 2*L*step)/(1-2*step*L) < 1)
    print((1/(m*c*step) + 2*L*step)/(1-2*step*L))
    for k in range(max_iter):
        # Computing the gradients at iteration k
        for k_ in range(n) :
            alpha[k_] = grad(x,k_)
        mean_alpha = np.mean(alpha,axis=0)
        if (k !=0):
            x_tab = np.vstack((x_tab, x))  
        # Updating the weights in the inner loop
        for m_ in range(m):
            i = np.random.randint(0,n)
            new_grad = grad(x,i)
            g = new_grad - alpha[i] + mean_alpha
            x = x - step * g
            if l1:
                x = prox(x, step * l1)
            if (k*m+m_)%n == 0 and (k*m+m_)!=0: # each completed epoch
                x_tab = np.vstack((x_tab, x))  

    return x, x_tab

Conclusions¶

Here we have shown empirically that for an easy problem with correlated data, stochastic gradient algorithms outperform Standard Gradient Descent. This is due in part to the redundancy and correlations present in the data.

We have also shown that gradient aggregation techniques such as SAGA and SVRG possess nice theoretical and empirical convergence for strongly convex objective whose gradient is Lipschitz continuous. This is partly due to their variance reduction properties. Nonetheless, we must remain aware that the computing time of those methods grows with the number of examples $n$. For very large $n$, these methods are likely to perform worse than Stochastic Gradient Descent.

Answer:

  • The minimization problem we consider is now : $$\begin{align*}\min_{x\in\mathbb{R}^d } F(x) := \underbrace{ \frac{1}{n} \sum_{i=1}^n \log( 1+\exp(-b_i \langle a_i,x \rangle) ) + \frac{l_2}{2} \|x\|_2^2}_{f(x)} + \underbrace{l_1 \|x\|_1 }_{g(x)}.\end{align*}$$
  • The objective function consists of two maps $f$ and $g$, the former has "good" properties, while the latter is non-differentiable.The map $F$ remains $\mu$ strongly convex but looses its $L$-smooth feature. We will deal with this non-differentiable term through a proximal operator $$prox_{\gamma g} \left( x\right) := \underset{v\in \mathbb{R}^d}{argmin} \: \gamma g(v) + \frac{1}{2} \Vert{v-x}\Vert_2^2$$
self._empirical_risk = lambda x: sum(self._logistic_loss(A, b, x))/n \
                                 + self.l2/2.0 * np.linalg.norm(x,2) \
                                 + self.l1 * np.linalg.norm(x,1)

The minimization algorithms we've implemented have a slow rate of convergence for this new objective function. We will therefore use proximal variations of these algorithms.

Question: Using the $\textrm{prox}_{l_1 \|\cdot\|_1}$ operator, write down the $k^{th}$ step of proximal GD, proximal SGD, proximal SAGA, and proximal SVRG.

Answer: For each of the algorithm we replace the step \begin{equation} x_{k+1} = x_k - \gamma_k \cdot g_k \end{equation} with \begin{equation} x_{k+1} = \textrm{prox}_{\gamma_k l_1 \|\cdot\|_1}( x_k - \gamma_k \cdot g_k ) \end{equation} where $\gamma_k$ is the step size and $g_k$ is the descent direction.

Question:
  • Give a closed-form expression of $\textrm{prox}_{l_1 \|\cdot\|_1}$
  • Complete TODO (9) (note that you can access the $l_1$ coefficient using self.l1).
  • Add your code to \lib\linear_model.py.

Answer:

  • We obtain $\textrm{prox}_{\gamma l_1 \|\cdot\|_1}$ by applying componentwise : \begin{equation} \textrm{prox}_{\gamma l_1 \left|\cdot\right|} \left(x_i \right) := \begin{cases} 
      x_i + \gamma l_1 & \text{ if } x_i \in \left]-\infty, -\gamma l_1 \right] \\
  x_i - \gamma l_1 & \text{ if } x_i \in \left[\gamma l_1 , +\infty \right[ \\
  0 & \text{ if } x_i \in \left[ -\gamma l_1 , \gamma l_1\right]
    \end{cases} \end{equation}

This operator is named the soft-thresholding operator.

Answer:

On the plot we can see that increasing the penalty on the $l_1$ norm of coefficients leads to increasingly sparser solutions. It also shrinks coefficients (though $l_2$ penalty is better for this purpose).

Possible use cases :

  • If we have collinear features, $l_1$ regularization is likely to enforce coefficients of some of those features to be zero.
  • In some applications such as embedded devices, having a small model may be crucial to satisfy constraints arising from hardware limitations.
  • We may want to reduce the number of necessary features if they are hard or expensive to acquire.
  • The less feature a model uses, the more interpretable it is.

ADMM¶

We offer an implementation of the alternating direction method of multipliers (ADMM). We write again our function to minimize $$\begin{align*} F(x) := \underbrace{ \frac{1}{n} \sum_{i=1}^n \log( 1+\exp(-b_i \langle a_i,x \rangle) ) + \frac{l_2}{2} \|x\|_2^2}_{f(x)} + \underbrace{l_1 \|x\|_1 }_{g(x)}.\end{align*}$$ The algorithm consists in iterating : \begin{eqnarray} x_{k+1} &= \text{prox}_{1/\rho f}\left( z_k - \frac{u_k}{\rho} \right) \\ z_{k+1} &= \text{prox}_{1/\rho g}\left( x_{k+1} + \frac{u_k}{\rho} \right) \\ u_{k+1} &= u_k + \rho \left(x_{k+1} - z_{k+1} \right) \end{eqnarray}

convergence of the algorithm is ensured thanks to the fact that $\partial f$ and $\partial g$ are both maximal monotone (see corollary 4.1 if "On the Douglas-Rachford Splitting Method and the Proximal Point Algorithm for Maximal Monotone Operators by Jonathan ECKSTEIN and Dimitri P. BERTSEKAS".) Indeed, $f$ and $g$ both are lower semi-continuous and proper (cf. proposition 12.b of "J.J. MOREAU Proximité et dualité dans un espace hilbertien Bulletin de la S. M. F., tome 93 (1965), p. 273-299")

Newton Method¶

We wanted to see how a (stochastic) second order method performs on this simple problem so we implemented standard and subsampled Newton Methods.

A Newton step is described by the following equation :

$$ x_{k+1} = x_k + \alpha_k s_k $$

where $s_{k}$ satisfies $\nabla^{2} F\left(w_{k}\right) s_{k}=-\nabla F\left(w_{k}\right)$.

Since it is (too) costly both in terms of computation and storage, several ideas have been proposed to reduce its cost such. Here we decided to implement Gradient and Hessian subsampling. In our implementation of the Subsampled Newton method, we sub-sampled both the gradient and the Hessian. In the subsampled case, we used Armijo line search.

Subsampled Newton Method¶

At each iteration :

  • we draw $|\mathbb{S}_G|$ samples to estimate the Gradient $\nabla \tilde{F}\left(w_{k}\right)$
  • we draw $|\mathbb{S}_H|$ samples to estimate the Hessian $\nabla^{2} \tilde{F}\left(w_{k}\right)$
  • we solve $\nabla^{2} \tilde{F}\left(w_{k}\right) s_{k}=-\nabla \tilde{F}\left(w_{k}\right)$ for $s_k$ using the previous estimates
  • we use backtracking (Armijo) line search to choose the step size $\alpha_k$

Some things to note :

  • $\mathbb{S}_G$ and $\mathbb{S}_H$ are conditionnally uncorelated given $w_k$
  • We solve the linear system exactly but we could use an inexact (iterative) solver such as Conjugate Gradient. In this case, we don't even have to compute the Hessian estimates since we can work with the Hessian-vector product. Unfortunately, we didn't have time to implement Subsampled Hessian Free Newton methods.
  • Since we don't have access to the true value of the gradient, Armijo line search may give a length $\alpha$ for which the observed cost decrease sufficiently whereas the true cost may increase or the observed cost may not decrease sufficiently whereas the true cost decreases sufficiently.

    ''' Standard Newton Method '''
    n = len(b)
    x = np.copy(x0)
    x_tab = np.copy(x)
    for k in range(1, max_iter):
        g = grad(x, i=None)
        H = hessian(x, i=None)
        s = np.linalg.solve(H, -g)
        x = x + s
        
        # Saving each iteration
        x_tab = np.vstack((x_tab, x))

    return x, x_tab

def S_NEWTON(x0, loss, grad, hessian, max_iter):
    ''' Subsampled Newton method with backtracking '''
    eta = 1 # Constant multiplier
    gamma = 0.8 # Step multiplier for backtracking
    n = len(b)
    grad_size = 50
    hess_size = 20
    
    x = np.copy(x0)
    x_tab = np.copy(x)
    for k in range(1, max_iter):
        idx_grad = np.random.choice(n, grad_size)
        idx_hessian = np.random.choice(n, hess_size) 
        
        g = grad(x, i=idx_grad)
        H = hessian(x, i=idx_hessian)
        s = np.linalg.solve(H, -g)
        
        step = 1
        while(loss(x + step * s) > loss(x) + eta * step * g @ s):
            step = step * gamma        
        x = x + step * s
        
        # Taking into account (approximatively) the hessian computations
        if k%(A.shape[0]//hess_size) == 0: 
            x_tab = np.vstack((x_tab, x))
    return x, x_tab

            '''
            Gradient of the objective function at x.
            If no i is given, grad should return the batch gradient.
            Otherwise, it should return the gradient associated to the samples (A[i], b[i]).
            '''
            if i is None: # return batch gradient
                output = -b/(1 + np.exp(b*np.dot(A, x)))
                output = sum(np.diag(output))/len(b) @ A
                output += self.l2*x
            else: # return partial gradient associated to the samples (A[i], b[i])

                output = sum(np.diag(output))/len(i) @ A[i]
                output += self.l2*x
            return output

        def hessian(x , i=None):
            '''
            Hessian of the objective function at x.
            If no idx is given, hessian should return the full Hessian.
            Otherwise, it should return the hessian associated to the samples (A[i], b[i]).
            '''
            if i is None: # return batch gradient
                S = 1/(1 + np.exp(-b*np.dot(A, x))) * (1 - 1/(1 + np.exp(-b*np.dot(A, x)))) / len(b)
                S = np.diag(S)
                output = A.T @ S @ A + self.l2 * np.eye(A.shape[1])

                S = 1/(1 + np.exp(-b[i]*np.dot(A[i], x))) * (1 - 1/(1 + np.exp(-b[i]*np.dot(A[i], x))))/len(i)
                S = np.diag(S)
                output = A[i].T @ S @ A[i] + self.l2 * np.eye(A.shape[1])
            return output

Notes on the Newton Method¶

We have shown that a simple implementation of subsampled Newton Method may give good results for our classification problem. In this approach, it is really important to choose "good" subsample sizes which are small enough for our computations to be fast and big enough for our computations to be accurate. Recently, some guarantees have been given for specific sampling strategies \textit{Sub-Sampled Newton Methods I: Globally Convergent Algorithms, Farbod Roosta-Khorasani and Michael W. Mahoney}. One other possible improvement for our method would be to change the step-size selection.

Considering the computational side of things, we could replace the exact linear solver by an inexact solver (using Conjugate Gradient for instance) to reduce the computational overhead. We could also exploit the particular form of our linear system to reduce the computational requirements by noting that there is a simple expression for the Hessian-vector product which doesn't require computing and using the full Hessian.

Besides there exist other Stochastic Inexact Newton Methods which have interesting properties such as Stochastic Quasi-Newton methods.

Conclusion¶

During this lab, we implemented several stochastic optimization algorithms and compared their performances using step sizes ensuring their good convergence. We then implemented $l_1$ regularization via the proximal operator for our algorithms and showed the effect of $l_1$ and $l_2$ regularization. Finally, we decided to investigate two additional methods, namely the alternating direction method of multipliers and the subsampled newton method. Those two methods complement nicely the previous work since they respectively allow us to tackle non differentiable problems and use approximate curvature information to improve the gradient descent.
