II. The optimization problem

Logistic regression consists in finding the parameter $x \in \mathbb{R}^d$ that best fits: $$ b_i = \begin{cases} \phantom{-}1 \ \textrm{ if }\ \langle a_{i}, x\rangle \ge 0 \\ -1 \ \textrm{ if }\ \langle a_{i}, x\rangle < 0 \end{cases} $$ for all the samples $i$ of the training set (where $a_i$ is the $i^{th}$ line of $A$ and $b_i$ the $i^{th}$ element of $b$). To measure this fit, logistic regression chooses the following loss:

$$ \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) = \log \left(1+\exp \left(-b_{i}\left\langle a_{i}, x\right\rangle\right)\right) $$

The corresponding optimization problem we will consider is the following:

Minimization of the empirical risk $f$ of the $L_2$-regularized logistic regression model, on the dataset $(A_{\textrm{train}},b_{\textrm{train}})$: $$ \begin{aligned} \textrm{min}_{x\in \mathbb{R}^d} \ f(x) \ &= \ \textrm{min}_{x\in \mathbb{R}^d} \ \frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} f_i(x) \\ &= \ \textrm{min}_{x\in \mathbb{R}^d}\frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) + \frac{l_2}{2}\|x\|^{2}_2, \end{aligned} $$ where $l_2 \ge 0$ is the $L_2$ regularization coefficient.

To simplify the notation, we'll stop indexing $A$, $b$ and $n$ with "$\textrm{train}$" in the following.

Question: $f$ is $\mu$-strongly convex.
  • Give a lower bound on $\mu$.
  • Complete TODO (1) in \lib\linear_model.py.

Answer:

$\forall k \in \{1,...n\}$

$\frac{\partial \textrm{LogisticLoss}(b_k, \left\langle a_{k}, x\right\rangle)}{\partial x_i} = \frac{-b_k.a_k^i \textrm{exp}(-b_k \langle a_k, x \rangle)} {1+ \textrm{exp}(-b_k \langle a_k, x \rangle)}$

$\frac{\partial^2 \textrm{LogisticLoss}(b_k, \left\langle a_{k}, x\right\rangle)}{\partial x_i \partial x_j} = \frac{a_k^i.a_k^j \textrm{exp}(-b_k \langle a_k, x \rangle)} {\left( 1+ \textrm{exp}(-b_k \langle a_k, x \rangle) \right)^2} \ge 0$

$x \mapsto \textrm{LogisticLoss}(b_k, \left\langle a_{k}, x\right\rangle)$ is convex

And $x\mapsto l_2\|x\|^2_2$ is $l_2$-strongly convex.

So $f$ is $l_2$-strongly convex, and $\mu \ge l_2$.

Question: Denote $\lambda_{\max}$ the largest eigenvalue of the real symmetric matrix $AA^T$.
  • Prove that $\lambda_{\max}(\nabla^2 f_i) \le \lambda_{\max}(\frac{a_i a_i^T}{4} + l_2 I_d)$
  • Prove that $\lambda_{\max}(a_i a_i^T) \le \|a_i\|^2_2$

Answer:

  • 1) $$\left[ \nabla^2 fk \right]{i,j}= \frac{a_k^i . a_k^j \textrm{exp}(-b_k \langle a_k, x \rangle)}{\left(1+\textrm{exp}(-b_k\langle a_k,x\rangle)\right)^2}
  • l2\delta{i,j} $$ $$\nabla^2 f_k = a_k a_k^T u(x) + l_2 I$$ where $u(x) = \frac{\textrm{exp}(-b_k \langle a_k, x \rangle)}{\left(1+\textrm{exp}(-b_k\langle a_k,x\rangle)\right)^2} $

and $$\lambda_\textrm{max}(\nabla^2 f_k) = \left(\|a_k\|^2 u(x)^2 + 2 . l_2 . u(x)\right) . \lambda_\textrm{max}(a_k . a_k^T) + l_2^2$ by the way, $u(x) \le \frac{1}{4}$$ and $$\lambda_\textrm{max}(\frac{a_k . a_k^T}{4} + l_2 I) = \left(\frac{\|a_k\|^2}{16} + \frac{l_2}{2}\right)\lambda_\textrm{max}(a_k . a_k^T) + l_2^2$$ then $$\lambda_\textrm{max}(\nabla^2 f_k) \le \left(\frac{\|a_k\|^2}{4} + 2 . \frac{l_2}{4}\right) . \lambda_\textrm{max}(a_k . a_k^T) + l_2^2 \le \lambda_\textrm{max}(\frac{a_k . a_k^T}{4} + l_2 I)$$

  • 2) $$ \begin{aligned} (a_i . a_i^T).a_i &= a_i.(a_i^T.a_i) &= \|a_i\|_2^2 a_i \end{aligned}$$ then 0 is eigen value of $a_i.a_i^T$ with multplicity $n-1$ and $\|a_i\|_2^2$ is eigen value with multiplicity 1. $$\text{then }\lambda_{\max}(a_i a_i^T) = \|a_i\|_2^2$$

A consequence of these results is that the $f_i$s and $f$ are all $L$-smooth with $L \le \frac{\max_i\|a_i\|^2_2}{4} + l_2$ and we can therefore use the following value in our implementation:

L = 0.25 * max(np.linalg.norm(A,2,axis=1))**2 + self.l2

def grad(x, i=None):
    '''
    Gradient of the objective function at x.
    If no i is given, grad should return the batch gradient.
    Otherwise, it should return the partial gradient associated the the datapoint (A[i], b[i]).
    '''

    if i is None: # return batch gradient
        output = -b/(1 + np.exp(b*np.dot(A, x)))
        output = sum(np.diag(output)).dot(A)/n
        output += self.l2*x
    else: # return partial gradient associated the datapoint (A[i], b[i])
        output = -b[i]/(1 + np.exp(b[i]*(np.dot(A[i], x))))
        output = output* A[i]
        output += self.l2 * x 

    return output

def GD(x0, grad, max_iter, n, L, mu):
    '''
    Parameters
    ----------
    x0: array, shape (nb_features,)
        Initialisation of the solver
    grad: function
        Gradient of the objective function
    max_iter: int
        Number of iterations (i.e. number of descent steps). Note that for GD or SVRG, 
        one iteration is one epoch i.e, one pass through the data, while for SGD, SAG and SAGA, 
        one iteration uses only one data point.
    n: int
        Dataset size
    L: float
        Smoothness constant of the objective function
    mu: float
        Strong convexity constant of the objective function
        
    Returns
    -------
    x: array, shape (nb_features,)
        final iterate of the solver
    x_tab: array, shape (nb_features, max_iter)
        table of all the iterates
    '''
    stepsize  = 2/L - 0.001  #We want to maximize the stepsize, but still be under 2/L, we choose 0.001 of tolerance
    x         = x0
    x_tab     = np.copy(x)

    for k in range(max_iter):
        x     = x - stepsize*grad(x)
        x_tab = np.vstack((x_tab, x))

    return x, x_tab

    x = x0

    for k in range(max_iter):
        stepsize = 0.1*np.log(k+2)**2/(k+1.0)
        i = random.randint(0,n-1)
        x = prox(x - stepsize*grad(x, i))
        
        if k%n == 0: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SAGA (Stochastic Average Gradient "Amélioré")¶

SAGA starts by computing a single batch gradient, then at iteration k, the following update is executed:

$$ \begin{cases} i \textrm{ chosen randomly in } \{1...n\}\\ x_{k+1} \leftarrow x_{k} - \gamma_k\big(\nabla f_i(x_k) - \alpha_i + \bar{\alpha}\big) \\ \alpha_i \leftarrow \nabla f_i(x) \end{cases} $$

where $(\alpha_j)_{j=1}^n$ are gradients evaluated at previous iterates and $\bar{\alpha}=\frac{1}{n}\sum^n_{j=1}\alpha_j$.

Answer: If f is L-lipzitch, strongly convex then the SAGA converge with stepsize $\frac{1}{2(\mu n + L)}$ [1]

In the non strongly convex case, a recommended stepsize is $\frac{1}{3L}$ [1]

For generality, we will use $\gamma = \frac{1}{3L}$

source : [1] SAGA: A Fast Incremental Gradient Method With Support for Non-Strongly Convex Composite Objectives (Aaron Defazio Francis Bach Simon Lacoste-Julien) https://arxiv.org/pdf/1407.0202.pdf

def SAGA(x0, grad, max_iter, n, L, mu):
    x = x0
    x_tab = np.copy(x)
    stepsize = 1.0/(2*(mu*n+L))
    tab = np.array([grad(x0, i) for i in range(n)])
    for k in range(max_iter):
        i = random.randint(0,n-1)
        g = grad(x,i)
        x = x - stepsize * (g - tab[i] + sum(tab)/n)
        tab[i] = grad(x,i)

        if k%n == 0: # each completed epoch
            x_tab = np.vstack((x_tab, x))

    return x, x_tab

SVRG (Stochastic Variance Reduced Gradient)¶

At iteration k of the SVRG algorithm, we perform the following:

$$ \begin{cases} \textrm{Compute the batch gradient } \nabla f(x_k)\\ \textrm{Initialise }y_0 = x_k\\ \textrm{for m in 0...M-1: } \begin{cases} i \textrm{ chosen randomly in } \{1...n\} \\ y_{m+1} \leftarrow y_{m} - \gamma_k\big(\nabla f_i(y_m) - \nabla f_i(x_k) + \nabla f(x_k)) \end{cases}\\ x_{k+1} \leftarrow y_{M} \end{cases} $$

def SVRG(x0, grad, max_iter, n, L, mu):
    x = x0
    x_tab = np.copy(x)
    stepsize = 1.0/(2.1*L)
    for k in range(max_iter):
        grad_batch = grad(x)
        y=x
        M=n
        for m in range(M):
            i = random.randint(0,n-1)
            y = prox(y- stepsize*(grad(y,i) - grad(x,i) + grad_batch))
        x=y
        x_tab = np.vstack((x_tab, x))
        
    return x, x_tab

Answer: Unlike SGD, the learning rate for SVRG does not have to decay, which leads to faster convergence as one can use a constant learning rate: $\gamma \in ] 0, \frac{1}{2L}[$ [2].

source: Accelerating Stochastic Gradient Descent using Predictive Variance Reduction (Rie Johnson, Tong Zhang) https://papers.nips.cc/paper/4937-accelerating-stochastic-gradient-descent-using-predictive-variance-reduction.pdf

Question:

  • Is this objective $\mu$-strongly convex, $L$-smooth, differentiable?
  • Complete the TODO (8) to account for the new term in the objective.
  • Add your code to lib/linear_model.py.

Answer:

i) Here is the empirical risk minimization objective with both $L_1$ and $L_2$ regularizations: $$ \begin{aligned} \textrm{min}_{x\in \mathbb{R}^d} \ f(x) &= \ \textrm{min}_{x\in \mathbb{R}^d}\frac{1}{n_{\textrm{train}}} \sum_{i=1}^{n_{\textrm{train}}} \textrm{LogisticLoss}(b_i, \left\langle a_{i}, x\right\rangle ) + \frac{l_2}{2}\|x\|^{2}_2 + l_1\|x\|_1, \end{aligned} $$

ii) $\|.\|_1$ is convex, so the objective function is $\mu$-strongly convex.

$\|.\|_1$ is differentiable almost everywhere, so is the objective function.

$\|.\|_1$ is not differentiable everywhere so the the objective function is not L-smooth.

self._empirical_risk = lambda x: sum(self._logistic_loss(A, b, x))/n \
                                 + self.l2/2.0 * np.linalg.norm(x,2) 
                                 + self.l1*np.linalg.norm(x,1)

The minimization algorithms we've implemented have a slow rate of convergence for this new objective function. We will therefore use proximal variations of these algorithms.

I) Optimal tuning of $l_1$ and $l_2$ parameters¶

We have seen, in the previous section that it can be interesting to use $L_1$ and $L_2$ regularizations. A first idea would be: from an initialised value $(l_1^0, l_2^0)$ we repeat the following untill convergence:

$$ \begin{cases} \textrm{Compute the score on the test set for a range of value for }l_1\textrm{, with }l_2\textrm{ fixed.}\\ \textrm{Select the value for }l_1\textrm{ where the score is maximum.}\\ \textrm{Compute the score on the test set for a range of value for }l_2\textrm{, with }l_1\textrm{ fixed.}\\ \textrm{Select the value for }l_2\textrm{ where the score is maximum.} \end{cases} $$

Here are plotted the values of the score for various value of $l_1$ and $l_2$. With this plot, we have, with $l_1=0.01$ fixed, the maximum score is obtain with $l_2=0.01$, on the other hand, when $l_2=0.01$ fixed, the maximum score is obtain with $l_1=0.01$ so we reached convergence of the above algorithm.

A popular quasi-Newton method : BFSG¶

In another section, we try a very popular quasi-Newton method: BFGS. From an initial $x0$ and an approximate Hessian matrix $H_0$, BFGS algorithm converge to the solution by repeating the following:

  1. $p_k \leftarrow -H^{-1}_k\nabla f(x_k)$

  2. perform a linear search : $\alpha_k \leftarrow \underset{\alpha \in \mathbb{R}}{\operatorname{argmin}}f(x_k + \alpha p_k)$

  3. $s_k \leftarrow \alpha_k p_k$, $x_{k+1} \leftarrow x_k+s_k$

  4. $y_k \leftarrow \nabla f(x_{k+1}) - \nabla f(x_k)$

  5. $H_{k+1} \leftarrow H_k +\frac{y_k^T y_k}{y_k^Ts_k} - \frac{H_k s_k s_k^T H_k}{s_k^T B_k s_k}$
